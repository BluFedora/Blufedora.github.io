Blog Post 3:
The Inner Working Of Image Processing

BY: Shareef Raheem | Date: 11/29/2017

Introduction:

This post will be on how image processing is done and what techniques are used to create certain effects such as a ‘Gaussian Blur'. I have digressed away from talking about image processing from a more technical perspective and want to come back to that. In the last blog post, I covered the philosophical debates of photography as this technology advances but not so much what advances were made. This was so I could observe the most direct effect imaging technology had on the world.

Although digital image processing is a broad topic, it happens to be “a subfield of signals and systems” (TutorialsPoint). Image processing at its core is the method of obtaining information from a particular signal; namely pictures. This information can be the detection of a face to unlock a device or given a set of points that shade a 3D model accordingly. For my purposes these signals are solely going to be grids of pixels that need to be changed or manipulated in some way.

Terminology and Images:

I’ve mentioned ‘pixels’ a couple of times throughout my blog posts but never fully defined them. A pixel is the smallest unit of an image, the value is simply an integer. These pixels are put in a 2 dimensional grid to represent some kind of image. Below is a table (Image A) representing some image of pseudo random ‘colors’.

101 203 231 213
53 200 25 31
21 0 255 2

(Image A: 4 rows x 4 cols = 16 pixels total)

All these numbers in ‘Image A’ are under the number 256; this is a conscious decision. This is a representation of an 8-bit image meaning each pixel must be represented by eight 1’s and 0’s (binary). This gives us from 0 to 255 to represent the intensity of that pixel on the image. The color black is always 0, while white is variable, always being the maximum number that a pixel can be. The number of colors a pixel can represent can be calculated with this formula:

Number of colors = (2)bits_per_pixel

Most color images use 24 bits per pixel to have enough space to represent the base three colors: red, green, and blue. Each color channel will have eight bits of accuracy:

PixelColor(24bits) = Red(8bits) + Green(8bits) + Blue(8bits)

Now we know the storage of an image depends on the amount of rows and columns along with the bits per pixel. The size of the image in bits is:

rows x cols x bit_per_pixel.

Methods on which to Change:

Images for the most part are manipulated in one of two ways, the first (affine transformations) keeps the data of the image intact (linearly speaking) while the other (convolution) warps the pixels for either an effect or to extract information such as edge.

Affine Transformations:

Affine transformations use transformation matrices to rotate, scale, reflect, sheer or translate an image. Although we don’t gain new information from these transformations they do have their use in photography and video games. For example, in the post-production phase of photography a mountain may be too small for the photographer so they will use some image editing tool to scale it up. The image editing tool most likely uses a scaling matrix to get the work done efficiently. Rather than working on a pixel by pixel basis this matrix would be applied to the four corners of the image.

(Table B: This matrix would scale an image by 2 horizontally and keep the same vertical height.)
Convolution Matrices:

These type of manipulation works on a pixel by pixel basis. It uses a special kind of matrix called a ‘kernel’ (filter array). A kernel is a square matrix with an odd number of rows and columns so that it has center. Each pixel is multiplied by the center element in the kernel, now with this kernel centered at this pixel multiply the corresponding pixels and add them all together. This is how the new color at that pixel is calculated of the new output image. One thing to note is that the generated image ends up being the same size as the original.

Conclusion:

This is a solid foundation towards understanding image processing and what is done to images to extract information. We use imaging technology in our daily life and I feel that it is important to know how the world around us works. Technology is so embedded into our life that I consider it essential to understand it even at a basic level.
